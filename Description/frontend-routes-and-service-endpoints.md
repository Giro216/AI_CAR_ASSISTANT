# Frontend routes and service endpoints

Ниже зафиксирована карта того, какие routes и endpoints нужны по текущему фронтенду проекта `Service_Frontend`.

Документ обновлён с учётом текущих архитектурных заметок из `Диплом.md` и `Issues.md`: в целевую схему добавлены `Notification Service`, `VIN handler`, `Pricing Service` и `TCO Service`.

Важно: сейчас фронтенд собран как одна страница с секциями и модальными окнами. Отдельного client-side router-а в коде нет, поэтому под `routes` здесь понимаются основные пользовательские сценарии и будущие URL, которые нужно поддержать через `API Gateway`.

## 1. Что уже видно во фронтенде

### Основные UI-сценарии
- Главная страница с шапкой, блоком AI-помощника, популярными авто и футером.
- Переход в каталог автомобилей.
- Открытие профиля пользователя.
- Открытие AI-чата.
- Добавление и удаление автомобиля из избранного.
- Фильтрация каталога по цене, типу кузова, топливу и коробке передач.
- Запросы по VIN и истории автомобиля.
- Напоминания об обслуживании.
    - Просмотр рыночных цен и расчёт TCO.

### Следствие для backend
- Нужен единый `API Gateway`, через который фронтенд будет ходить во все сервисы.
- Каталог машин должен жить отдельно в `Service_Car_Catalog`.
- Профиль и избранное должны жить в `Service_User_Info`.
- Чат и подбор по запросу пользователя должны жить в `Service_Recommendation_LLM` или `Service_Chat_Bot`.
- VIN-операции лучше вынести в `Service_VIN_Handler`.
- Напоминания лучше вынести в `Service_Notification`.
- Рыночные цены лучше вынести в `Service_Pricing`.
- Расчёт полной стоимости владения лучше вынести в `Service_TCO`.

## 2. Frontend routes, которые стоит поддержать

Даже если сейчас приложение не использует React Router, эти маршруты стоит заложить на уровне API и при желании потом отразить в UI.

### Public routes
- `/` — главная страница.
- `/catalog` — каталог автомобилей.
- `/profile` — профиль пользователя.
- `/chat` — интерфейс AI-помощника.
- `/favorites` — избранные автомобили.
- `/vin` — поиск автомобиля по VIN.
- `/notifications` — напоминания и уведомления.
- `/pricing` — цены и история изменения цен.
- `/tco` — калькулятор полной стоимости владения.

### Если нужен более granular UI
- `/catalog/:id` — страница карточки автомобиля.
- `/recommendations` — страница результатов подбора.
- `/vin/:vin` — страница результата VIN-декодера.
- `/cars/:id/history` — история по конкретной машине.
- `/compare` — сравнение нескольких автомобилей.

## 3. API Gateway

`Service_API_Gateway` должен стать точкой входа для фронтенда.

### Базовые routes gateway
- `GET /api/v1/health` — проверка доступности gateway.
- `GET /api/v1/catalog/*` — проксирование запросов в `Service_Car_Catalog`.
- `GET /api/v1/users/*` — проксирование запросов в `Service_User_Info`.
- `GET /api/v1/chat/*` — проксирование запросов в `Service_Recommendation_LLM` или `Service_Chat_Bot`.
- `GET /api/v1/recommendations/*` — проксирование рекомендаций по автомобилям.
- `GET /api/v1/vin/*` — проксирование VIN-декодинга и связанных данных.
- `GET /api/v1/notifications/*` — проксирование напоминаний и уведомлений.
- `GET /api/v1/pricing/*` — проксирование рыночных цен и истории цен.
- `GET /api/v1/tco/*` — проксирование расчёта TCO.

### Желательно добавить общие вещи
- Авторизацию через Keycloak.
- CORS для фронтенда.
- Единый формат ошибок.
- Агрегацию данных для главной страницы и профиля.

## 4. Service_Car_Catalog

Этот сервис отвечает за каталог, карточки автомобилей, фильтры, сортировку и выдачу популярных авто.

### Routes и endpoints
- `GET /api/v1/cars` — список автомобилей с фильтрами и сортировкой.
- `GET /api/v1/cars/{car_id}` — подробная карточка автомобиля.
- `GET /api/v1/cars/popular` — популярные автомобили для блока `PopularCars`.
- `GET /api/v1/cars/search` — поиск по названию, бренду, модели.
- `GET /api/v1/cars/filters/meta` — справочники для фильтров: тип кузова, тип топлива, коробка передач, бренды.
- `POST /api/v1/cars/filter` — если фильтрация будет через body, а не query params.
- `GET /api/v1/cars/{car_id}/similar` — похожие автомобили для блока рекомендаций.
- `GET /api/v1/cars/{car_id}/pricing` — текущая цена и ценовая история по автомобилю, если pricing будет агрегироваться в каталоге.

### Что это покрывает во фронтенде
- `PopularCars`.
- `CatalogSection`.
- Кнопка `Подробнее` в карточке.
- Сортировку и фильтры.

### Минимальный контракт ответа
- `id`
- `brand`
- `model`
- `price`
- `year`
- `mileage`
- `engine`
- `transmission`
- `fuel`
- `bodyType`
- `imageUrl`
- `isNew`
- `isPopular`

## 5. Service_Notification

Этот сервис отвечает за напоминания об обслуживании, страховку, сезонную замену шин и другие уведомления.

### Routes и endpoints
- `GET /api/v1/notifications` — список уведомлений пользователя.
- `POST /api/v1/notifications` — создать напоминание.
- `PATCH /api/v1/notifications/{id}` — изменить напоминание.
- `DELETE /api/v1/notifications/{id}` — удалить напоминание.
- `GET /api/v1/notifications/templates` — шаблоны напоминаний: ТО, масло, шины, страховка.
- `POST /api/v1/notifications/test` — тестовое уведомление для проверки канала доставки.

### Что это покрывает во фронтенде
- Экран или модальное окно напоминаний.
- Настройки профиля, связанные с пушами и сервисными уведомлениями.
- Будущие карточки обслуживания автомобиля.

## 6. Service_VIN_Handler

Этот сервис нужен для VIN-декодинга, истории автомобиля и дополнительных данных по конкретному транспортному средству.

### Routes и endpoints
- `GET /api/v1/vin/{vin}` — декодирование VIN.
- `GET /api/v1/vin/{vin}/history` — история автомобиля по VIN.
- `GET /api/v1/vin/{vin}/listings` — связанные объявления и варианты.
- `GET /api/v1/vin/{vin}/photos` — фотографии по VIN.
- `POST /api/v1/vin/decode` — декодирование VIN из body, если удобнее отправлять JSON.

### Что это покрывает во фронтенде
- Страницу VIN-поиска.
- Будущую карточку истории автомобиля.
- Связку с экранами просмотра конкретного авто.

## 7. Service_User_Info

Этот сервис нужен для профиля, избранного, пользовательских настроек и, в дальнейшем, напоминаний.

### Routes и endpoints
- `GET /api/v1/users/me` — текущий профиль пользователя.
- `PUT /api/v1/users/me` — редактирование профиля.
- `GET /api/v1/users/me/favorites` — список избранных автомобилей.
- `POST /api/v1/users/me/favorites` — добавить автомобиль в избранное.
- `DELETE /api/v1/users/me/favorites/{car_id}` — удалить автомобиль из избранного.
- `GET /api/v1/users/me/settings` — настройки профиля и уведомлений.
- `PUT /api/v1/users/me/settings` — изменить настройки.

### Если понадобятся напоминания
- `GET /api/v1/users/me/reminders` — список напоминаний.
- `POST /api/v1/users/me/reminders` — создать напоминание.
- `PATCH /api/v1/users/me/reminders/{id}` — изменить напоминание.
- `DELETE /api/v1/users/me/reminders/{id}` — удалить напоминание.

### Что это покрывает во фронтенде
- `ProfileDialog`.
- Иконку сердца в `PopularCars` и `CatalogSection`.
- Будущие настройки профиля и напоминания.

### Минимальный контракт ответа профиля
- `id`
- `fullName`
- `email`
- `phone`
- `birthDate`
- `city`
- `createdAt`
- `favoritesCount`

## 8. Service_Recommendation_LLM / Service_Chat_Bot

Здесь лучше держать чат-подбор автомобиля, подсказки и генерацию рекомендаций.

### Routes и endpoints
- `POST /api/v1/chat/messages` — отправить сообщение пользователем и получить ответ ассистента.
- `GET /api/v1/chat/history` — история чата пользователя.
- `POST /api/v1/recommendations/car-search` — получить подборку автомобилей по пожеланиям.
- `POST /api/v1/recommendations/explain` — объяснить, почему авто подходит пользователю.
- `GET /api/v1/recommendations/suggestions` — быстрые запросы или шаблоны вопросов.

### Что это покрывает во фронтенде
- `AIChatSection`.
- `AIChatDialog`.
- Быстрые запросы внутри чата.

### Полезный формат запроса для чата
- `message`
- `userId`
- `context`
- `filters`
- `selectedCarIds`

## 9. Service_Pricing

Этот сервис нужен для рыночных цен, истории изменения цен и статистики по моделям.

### Routes и endpoints
- `GET /api/v1/pricing/cars/{car_id}` — текущая рыночная цена автомобиля.
- `GET /api/v1/pricing/cars/{car_id}/history` — история цены.
- `GET /api/v1/pricing/brands/{brand}` — статистика по ценам марки.
- `GET /api/v1/pricing/search` — поиск по ценам, если нужен отдельный pricing search.

### Что это покрывает во фронтенде
- Страницу цен и рыночной статистики.
- Блоки «цена новой» и «цена на вторичке».
- Будущие фильтры по цене и динамике рынка.

## 10. Service_TCO

Этот сервис нужен для расчёта полной стоимости владения автомобилем.

### Routes и endpoints
- `POST /api/v1/tco/calculate` — расчёт TCO по автомобилю и параметрам пользователя.
- `GET /api/v1/tco/cars/{car_id}` — базовый TCO по конкретной модели.
- `POST /api/v1/tco/simulate` — сценарный расчёт, если пользователь меняет пробег, срок владения или стиль езды.
